WORKFLOW 1: User Ticket Booking
Description:
This workflow explains how a user books an event ticket through the system.
Steps:
User opens the web application.
User signs up or logs into their account.
User browses the list of available events.
User selects a specific event.
Event details (date, venue, price, seats) are displayed.
User selects number of tickets.
System checks seat availability
User proceeds to online payment
Payment is processed successfully.
System confirms the booking.
A unique QR code ticket is generated.
Confirmation email is sent to the user.


Key Features Involved:
User authentication
Event browsing
Online payment
Seat management
QR code generation
Email notification

WORKFLOW 3: QR Code Scanning & Attendance
Description:
This workflow explains how attendance is marked using QR code tickets.
Steps:
User arrives at the event venue.
Organizer opens QR scanner in system.
User presents QR code ticket.
Organizer scans the QR code.
System retrieves ticket data.
System verifies ticket authenticity.
If valid → entry is allowed.
Attendance is marked automatically.
Seat count is updated.
If QR is scanned again → duplicate entry is blocked.
Key Features Involved:
QR code validation
Attendance tracking
Real-time seat management
Fraud prevention
